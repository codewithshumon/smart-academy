   - Islamic content review (for Islamic studies)

   - Subject matter expert approval
   - Administrative approval
   - Final sign-off

   - Upload to CMS
   - SEO optimization
   - Cross-linking
   - Quality check

   - Regular updates
   - Accuracy verification
   - Performance monitoring
   - User feedback integration

### Content Guidelines

#### General Guidelines
- Write in clear, simple language
- Use active voice
- Break content into scannable sections
- Include relevant keywords naturally
- Add meta descriptions
- Optimize for mobile viewing
- Ensure accessibility (WCAG 2.1)

#### Islamic Content Guidelines
- Verify all Quranic verses and Hadith references
- Use authentic sources
- Include Arabic text with English translation
- Follow proper Islamic terminology
- Review by qualified Islamic scholars
- Respect Islamic sensitivities

#### Technical Guidelines
- Use Markdown format (.md files)
- Follow naming conventions
- Include version control information
- Add last updated dates
- Use consistent formatting
- Include table of contents for long documents

#### SEO Guidelines
- Research keywords before writing
- Include keywords in headings
- Write compelling meta descriptions
- Use descriptive URLs
- Add alt text to images
- Create internal links

## Technical Implementation Notes

### File Organization
```
/content
  /public-website
    /homepage
    /about
    /academics
    /admissions
    /student-life
  /portals
    /parent
    /student
    /teacher
    /admin
  /policies
  /templates
  /help
  /legal
  /media
```

### Metadata Format
Each content document should include:
```markdown
---
title: Document Title
category: Category Name
subcategory: Subcategory Name
author: Author Name
created: YYYY-MM-DD
updated: YYYY-MM-DD
version: 1.0
status: draft|review|approved|published
keywords: keyword1, keyword2, keyword3
language: en|bn
---
```

### Version Control
- Use Git for version control
- Follow semantic versioning (MAJOR.MINOR.PATCH)
- Maintain changelog for each document
- Archive old versions

### Multi-language Support
- Maintain parallel documents for English and Bengali
- Use language codes in filenames (e.g., `about-us-en.md`, `about-us-bn.md`)
- Ensure translation consistency
- Maintain terminology glossary
